# User Stories - Antique Bookstore

## 1. Introduction

This document captures preliminary user stories describing expected interactions with the system from the perspective of key roles. 
User stories complement the SRS by expressing goals, motivations, and operational constraints.

## 2. User Roles Overview

* **Manager** — full operational and administrative control.
* **Assistant Manager** — supports the manager; has elevated access.
* **Salesperson** — handles daily sales and order processing.
* **System** — enforces business rules.

## 3. User Stories

### 3.1 Manager

1. As a manager, I want to access complete data about all books so that I can monitor inventory and maintain accurate records.
2. As a manager, I want to add and edit book information so that pricing, condition, and metadata remain up to date.
3. As a manager, I want to add, edit, and delete author records so that author information remains accurate.
4. As a manager, I want to view sales reports so that I can identify popular books and plan procurement.
5. As a manager, I want to view all orders so that I can control processing and ensure timely delivery.
6. As a manager, I want to add and edit employee records so that staff information remains current.
7. As a manager, I want to manage employee positions and responsibilities so that correct roles and permissions are assigned.
8. As a manager, I want to see sales statistics per employee so that I can assess performance.
9. As a manager, I want to update the status of orders so that I can ensure proper workflow completion.
10. As a manager, I want to manage customer data so that the customer base remains accurate.

### 3.2 Assistant Manager

1. As an assistant manager, I want to add and edit orders so that I can support the manager in order processing.
2. As an assistant manager, I want to view employee data so that I can assist with administrative tasks.
3. As an assistant manager, I want to view book and author information so that I can answer customer questions.
4. As an assistant manager, I want to view sales statistics so that I can support reporting and trend analysis.

### 3.3 Salesperson

1. As a salesperson, I want to access book information so that I can quickly answer customer inquiries.
2. As a salesperson, I want to create new orders so that I can record purchases efficiently.
3. As a salesperson, I want to update order status after payment so that the sales process is completed correctly.
4. As a salesperson, I want to add delivery information so that customer orders can be fulfilled.
5. As a salesperson, I want to access customer information so that I can assist repeat customers.
6. As a salesperson, I want to mark books as sold so that inventory remains accurate.

### 3.4 Book (Non-functional)

* *As a book, I want to be purchased and read so that I fulfill my purpose.*

## 4. Notes

* These user stories are **not exhaustive** and will evolve through discovery and iteration.
* Each story can later be broken down into **acceptance criteria**, which will serve as a basis for writing tests.
